# 在绿联NAS上搭建MyLife博客自动化工作流

2026年05月12日 19:06

## 🎯 今天做了什么

今天在绿联 DH4300PLUS NAS 上成功搭建了一套完整的 MyLife 博客自动化工作流。

## 📦 主要工作

### 1. **环境搭建**
- 清理了之前混乱的测试文件
- 在 Docker 容器中建立了干净的工作目录
- 配置了 v2raya 代理解决 GitHub 访问问题

### 2. **工作流开发**
- 创建了 `mylife-workflow.py` 脚本（337行完整代码）
- 实现了完整的博客文章生成、预览、确认流程
- 区分了三种内容类型：
  - 知识库文档（docs/）
  - 博客文章（blog/）
  - 碎碎念（blog/，带 authors 字段）

### 3. **Git 集成**
- 初始化了 Git 仓库
- 配置了远程仓库指向 GitHub
- 实现了自动提交和推送功能

### 4. **代理配置**
- 使用 v2raya 作为代理服务
- 端口：47891
- 成功解决了 GitHub 克隆问题

## 🔧 技术栈

- **NAS**: 绿联 DH4300PLUS (ARM架构)
- **容器**: Docker 26.1.0
- **代理**: v2raya (端口 47891)
- **博客**: Docusaurus 3.9.2
- **语言**: Python 3.13.5
- **版本控制**: Git

## 🚀 工作流特点

1. **类型区分明确**
   - 知识库：纯技术文档，无作者字段
   - 博客：技术文章，有分类标签
   - 碎碎念：个人随笔，必须有 `authors: [itqzy]`

2. **确认机制**
   - 每次生成内容后显示前25行预览
   - 用户必须确认（发布/编辑/取消）才能继续

3. **Git 优先**
   - 完全使用 Git 命令操作
   - 避免 GitHub API 依赖
   - 支持代理环境

## 🤔 感想

在 NAS 上做开发确实有些限制（ARM架构、网络问题），但最终通过 v2raya 代理和分步克隆解决了所有问题。

现在只需要运行 `python3 mylife-workflow.py` 就可以：
1. 选择内容类型
2. 输入主题
3. 生成内容
4. 预览确认
5. 自动提交
6. 推送到 GitHub

Cloudflare Pages 会自动部署，实现完整的 CI/CD 流程。
